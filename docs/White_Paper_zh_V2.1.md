# **《多元宇宙分叉激励模型白皮书（V2.1 版）》**

## **0. 引言：核心理念与法律基石**

本白皮书旨在为基于 **GitHub + P2P** 的去中心化、开放式内容创作项目提供一套全新的、以**数据为裁判**的激励和收益分配框架。本系统确保了内容的**极致开放、透明和公平**。该模式完全根据 **DIPCNPO 章程**的规定运作并受其约束。

### **0.1 核心理念**

* **市场自治（Market Autonomy）：** 内容的价值由社区的**引用次数**和**净赞数**决定，而非任何委员会或个人主观评定。
* **分叉即权利（Forking as Rights）：** 贡献者必须接受其创作在被**采纳路径**中可能被**修改、终结或移除**，且无权阻止。
* **法律外包（Legal Outsourcing）：** 本系统不承担对**剽窃、恶意破坏**等法律范畴的争议仲裁责任。此类争议完全由**受害方**通过法律途径解决。

### **0.2 贡献者许可协议（CLA）**

**【绝对前提】** 所有贡献者必须签署**贡献者许可协议（CLA）**，并**不可撤销地**声明：

1.  **永久许可：** 所有内容被永久、不可撤销地授权给DIPCNPO使用。
2.  **命运放弃：** 贡献者必须接受其创作的角色、设定或故事情节，在被**采纳路径**中**可以被后续作者修改或终结**，并放弃对该角色的**“生存权”和“最终命运”**的控制权。

## **1. 核心架构：多元宇宙分叉模型**

本模型基于 **GitHub + P2P** 的分布式版本控制特性，每个作品都是一个可无限分叉、可交叉引用的网状结构。

### **1.1 内容创作**

1.  **创作形式：** 创作者在自己的设备中创建和原作者同名的作品仓库，并在其中创作和更新内容。
2.  **内容独立：** 每个创作者的内容都是一个自洽的 **“平行宇宙”**或其一部分，通过文件链接的形式和其他同名作品仓库的内容互动。
3.  **作者责任：** 创作者必须保证内容的原创性，如果发现侵权，所有法律责任由其本人承担。作者创作的图像，不论内容，不分大小，每张按照100字来计算。作者创作的音乐，同样不分长度，每段也按照100字计算。暂不支持视频内容。
4.  **文件格式：** 文本文件只支持md格式，图像文件只支持jpg和png格式，音频文件只支持mp3格式。
5.  **读者权利：** 每个人既是作者又是读者，在阅读其他人的作品时有权**点赞**、**点踩**和**路过**。为了不影响读者的主观判断，所以不能看到其他读者对本文的投票结果，只有作者能看到，但他不能投票。由于作者有权修改已经发布的内容，所以不保证用户在每次进入相同页面都看到相同内容，因此可以随时修改自己的看法。读者还可以追溯看到所有文件的历史版本。

### **1.2 内容的引用与价值锚定**

* *唯一文件名：** 所有内容片段（包括但不限于：故事章节、图像、音频，或角色、场景、道具、物种、魔法、武功、门派、单位、法则、现象、理念、信仰的定义等）都必须存放在独立的文件中，文件中可链接到其他文件。
* **显式引用：** 作者在创作时，通过 **文件链接显式引用** 自己或其他作者分支中的文件，形成网状结构。如果要在原作者的文件中添加对自己文件的引用，必须向原作者提出申请，原作者可以同意也可以拒绝。在md文件中的引用格式为`[文本](版本号/用户名/作品/路径/文件名.扩展名)`，版本号可以省略，默认显示最新版本。
* **价值定义：** 内容的价值不由人为决定，而是由以下 **三个客观数据** 决定其价值：**被采纳路径中的有效字数、被引用次数、净赞数**。

## **2. 激励与收益计算**

贡献者从**创作者资助基金**中获得分配的资格和权重完全基于**内容在被商业化路径中产生的客观数据**决定。

### **2.1 贡献积分公式**

贡献者的总积分（即收益分配权重）由以下**三个核心数据指标**加权计算：

贡献积分 = 路径中所有文件的积分总和

单文件积分 = （字数权重 * 基础字数分 + 引用权重 * 引用次数） * （本文件净赞值/所有文件净赞值）

| 指标 | 代表价值 | 计算方式 |
| :--- | :--- | :--- |
| **基础字数分** | **投入的努力（体量）** | 贡献者**在最终被采纳的商业化路径中**的有效新增字数总和。 |
| **引用次数** | **内容的基石性（实用性）** | 贡献者的内容**在最终被采纳的商业化路径中**被其他作者**有效引用的次数总和**。 |
| **净赞值** | **社区的喜好（质量）** | 贡献者的内容**在最终被采纳的商业化路径中**获得的 **净赞数** (赞数 - 踩数)。 |

### **2.2 动态权重的确定**

**作品理事会在商业化时协商投票决定权重**

1.  **协商：** 所有参与本次收益分配的作者（即内容被采纳路径中的所有作者）首先进行协商，提出自己对权重的建议。
2.  **投票:** 最终权重由**所有参与本次资助分配的作者**进行投票确定，以实现**受益者自治**。

### **2.3 商业化情景与分配**

资助分配基于**商业化产品实际采纳的内容路径**进行：
假定一个作品有ABCD四个创作者参与

1.  **线性产品（如电影）：** 仅计算**最终被采纳的、线性的**路径（例如 ACD 路径）中所有作者的贡献积分。
2.  **开放产品（如开放世界游戏）：** 计算**所有被纳入产品**的贡献者（A、B、C、D）的贡献积分。

## **3. 治理与法律边界**

本模型采用了**极简自治**的结构，设立**平台理事会**和**作品理事会**作为作品管理和战略决策机构。

### **3.1 平台理事会（Platform Council）的核心职能**

1. 仅专注于**商业化战略和 IP 法律风险**，制定白皮书，开发升级DIPCP，并履行受托责任。
2. CLA的统一管理。
3. **IP 法律风险传递：** 平台理事会**不进行**剽窃和版权冲突的认定。若出现此类法律问题，平台理事会有责任将**举报信息正式记录并告知涉及方**，但不参与法律仲裁或执行，完全交由**外部法律程序**解决。
4.  **内容监管最终权限:** 根据DIPCNPO章程第7C条，平台理事会（董事会）拥有最终权限，在获得75%绝对多数票通过时，可移除被合理认定属于"禁止用途"的内容（如煽动暴力、剥削性内容等），以保护平台及DIPCNPO的法律存续。
5.  按照项目理事会协商表决的结果，从**创作者资助基金**中向获得提名的作者发放资助。

### **3.2 作品理事会（Creation Council）的核心职能**

1. **商业化路径决策：** 作品理事会是唯一拥有如何商业化，和决定采纳哪条分叉路径进行商业化的权力机构。
2.  **资助提名与投票组织:** 作品理事会基于贡献积分和社区影响力提名高价值创作者获得**创作者资助基金**的资助资格。理事会成员商议好2个权重的比例后，需发起投票，所有在有效路径中的作者都可以参与投票，最终资助接收人及分配金额由项目范围内的社区投票决定。

## **4. 应用举例**

### 4.1 基本算法
1. 假定A作者作为发起人写了一个篇关于**轩辕王朝**的作品。其中有**张三**和**李四**两个人物。那么就需要为这三个定义分别创建三个文件。**轩辕王朝**作为首页，其中要包括**张三**和**李四**这两个文件的链接。并且之后在每个章节中，每次出现这个三个名词，就会自动生成可跳转的链接，算一次引用。
2. 然后B作者看到了A的这个故事，觉得有趣，决定参与，他创建了**王五**这个角色，并要求A将**王五**的文件链接加入**轩辕王朝**的文件中。然后他写了**王五**的自传，但是没写完，而A也在他后续的作品中用到了**王五**这个角色。
3. 接着C作者，看到没结局的**王五**就帮他写了一个结局。
4. 而D作者，觉得C写的结局不好，又写了另外一个版本，没想到这个结局得到超多的点赞，引来电影公司要购买版权。
5. 由于这个故事路径中没有C作者什么事，所以理事会的成员就由**ABD**三人组成，协商如何设定**2个权重**的分配。
6. 假定整个作品中**轩辕王朝**引用了100次，文字量为5000字，800赞，**张三**和**李四**各10次引用，2500字，100赞，**王五**50次引用，10万字，500赞，D写的结局没有被引用，2万字，但有5000赞。
7. 那么A肯定希望**引用权重**高，B和D则希望**字数权重**高。
8. 于是三人协商决定：**引用权重** 40%，**字数权重** 60%。
9. 这个版权卖出100万：
- A的积分为：
	**轩辕王朝** ((100/160) * 40% + (0.5/13) * 60%) * (800/6500) = 0.0336094674556213
	**张三** ((10/160) * 40% + (0.25/13) * 60%) * (100/6500) = 0.0005621301775147929
	**李四** ((10/160) * 40% + (0.25/13) * 60%) * (100/6500) = 0.0005621301775147929
- B的积分为：((50/160) * 40% + (10/13) * 60%) * (500/6500) = 0.0451183431952663
- D的积分为：((0/160) * 40% + (2/13) * 60%) * (5000/6500) = 0.0710059171597633
- 总积分：0.1508579881656805
10. 最终分红：
- A = (0.0336094674556213+0.0005621301775147929*2) / 0.1508579881656805 * 1000000 = 230,241.22
- B = 0.0451183431952663 / 0.1508579881656805 * 1000000 = 299,078.25
- C = 0.0710059171597633 / 0.1508579881656805 * 1000000 = 470,680.52
11. 随后这个IP又被游戏公司看中，要开发一个开放世界的游戏，用到所有的故事内容，那这次C就也有份分成了，算法公式同上。

### 4.2 内容孤岛
1. 假定B作者看了A作者的故事后，写了一篇番外，但是他向A提交链接申请被拒绝，那么他的这篇文章就属于内容孤岛，使用创作平台的其他用户根本无法发现他的作品，计算整体IP价值的时候也不被算在其中。
2. 但由于他可以通过其他渠道将他的文章链接分享给朋友，假定有C作者又在B的基础上写了另一篇文章，并引用了B的文章，之后C的文章被已经接入主网中的某位作者引用，于是B的文章也就间接的接入了主网，由此产生了IP价值。

### 4.3 人格分裂
1. 假定AB两个作者都创作了名为**张三**的角色，分别保存在各自的目录中，作者C如果只读过B的文章，在后续创作中引用**张三**，那自然是B创作的张三。
2. D作者同时看过AB两人的文章，他的文章中提到**张三**时就会出现人格分裂的情况，他就必须选择一个他认为合适的人格来引用，当然他还可以自己创建一个新的**张三**，这些**张三**都同时存在于这个多元世界中。
3. E作者看了他们三人的文章，觉得很有趣，也写了一篇文章，将三个**张三**都链接到了一起，这样分叉的世界就又交叉到了一起。

### 4.4 内容补全
1. 假定A作者写了一篇文章，其中提到了一个路人甲，但并没有为他创建角色定义文件。B作者看完他的故事之后，觉得这个路人甲很有趣，就为他写了一篇角色定义。然后向A发出链接申请，A同意后，B的补全内容就被纳入了整个网络，以后其他作者提到路人甲的时候，自然就引用了B的文章。
2. 另一种情况，A在创作时有个逻辑漏洞，B发现了，就写了一篇文章帮他把这个坑填上，然后通知A，A在修订版中将B写的内容补充进去，并且链接到B的原文，并表示感谢，将来在计算字数、引用数和净赞数的时候，都会算上B的那一份。

### 4.5 抄袭判定
文章内容是否属于抄袭这非常难于判断，本系统完全依赖社区的市场机制和外部法律程序进行裁决和惩罚。

1. 市场惩罚（社区自治）：

举证与宣传： 受害者或读者有责任举证并向社区提出确实的证据，证明某篇文章（或文件）的大部分内容是抄袭的。

惩罚机制： 社区对抄袭内容的唯一惩罚是市场选择：通过大量的“踩”来表达不认可。这会直接导致该文件的净赞值为负，计算贡献积分为零，在商业化时被自动排除在采纳路径之外。

2. 法律责任与移除：

法律行动： 涉及版权、剽窃等法律行为，完全依赖受害方通过外部执法机构和法律程序解决。

理事会职责： 理事会无权强制移除内容，但有权移除作品在DIPCP上的链接。一旦收到有法律效力的裁决（如法院禁令），理事会将通知所有作者，并在商业化路径决策中，永远排除该文件及其所处的路径。

3. 引用规范：

复制的例外： 假定 A 作者在 张三 的文件中已经详细设定了他的容貌、性格等诸多特点，而 B 作者在一篇《张三》传记中，因为情节需要，直接复制了其中部分或全部的内容。只要像 4.4.2 中那样加上感谢链接，这种情况就不算作抄袭，而视为对源内容高价值的引用，这会为被引用方（A）带来**“引用次数”**的积分增益。
