# **「多元宇宙フォークインセンティブモデル白書（V2.1版）」**

## **0. 序文：核心理念と法的基盤**

本白書は、**GitHub + P2P** に基づく分散型・オープンコンテンツ制作プロジェクトに対し、**データを審判とする**新しいインセンティブと収益分配の枠組みを提供することを目的とする。本システムは、コンテンツの**極致の開放性、透明性、公平性**を保証する。このモデルは **DIPCNPO 定款** の規定に完全に従って運用され、その制約を受ける。

### **0.1 核心理念**

* **市場自治（Market Autonomy）：** コンテンツの価値は、いかなる委員会や個人の主観的評価ではなく、コミュニティによる**引用回数**と**純「いいね」数**によって決定される。
* **フォークの権利化（Forking as Rights）：** 貢献者は、自身の制作物が**採用パス**内で**修正、終了、または削除**される可能性があることを受け入れ、それを阻止する権利を有しない。
* **法的業務の外部委託（Legal Outsourcing）：** 本システムは、**盗用、悪意のある破壊**などの法的領域における争議の仲裁責任を負わない。このような争議は、**被害者**が法的経路を通じて完全に解決するものとする。

### **0.2 貢献者ライセンス同意書（CLA）**

**【絶対的前提】** すべての貢献者は、**貢献者ライセンス同意書（CLA）** に署名し、以下のことを**取消不能に**宣言しなければならない：

1.  **永久的許諾：** すべてのコンテンツは、DIPCNPOが使用するために、永久的かつ取消不能に許諾される。
2.  **運命の放棄：** 貢献者は、自身の制作物の役割、設定、または筋書きが、**採用パス**内で**後の作者によって修正または終了される可能性がある**ことを受け入れ、そのキャラクターの **「生存権」と「最終的な運命」** に対する管理権を放棄する。

## **1. 核心アーキテクチャ：多元宇宙フォークモデル**

このモデルは、**GitHub + P2P** の分散型バージョン管理特性に基づいており、各作品は無限にフォークでき、相互参照可能な網状構造である。

### **1.1 コンテンツ制作**

1.  **制作形態：** 制作者は自身のデバイス上で、原作者と同じ名前の作品リポジトリを作成し、その中でコンテンツを制作・更新する。
2.  **コンテンツの独立：** 各制作者のコンテンツは、自律した**「パラレルワールド」**またはその一部であり、ファイルリンクの形式を通じて他の同名作品リポジトリのコンテンツと相互作用する。
3.  **作者の責任：** 制作者はコンテンツの独自性を保証しなければならない。権利侵害が発見された場合、すべての法的責任は本人が負う。作者が制作した画像は、内容やサイズにかかわらず、1枚あたり100語として計算する。同様に、作者が制作した音楽も、長さにかかわらず、1曲あたり100語として計算する。動画コンテンツは現在サポートしていない。
4.  **ファイル形式：** テキストファイルはmd形式のみ、画像ファイルはjpgとpng形式のみ、音声ファイルはmp3形式のみをサポートする。
5.  **読者の権利：** すべての人は作者であり同時に読者である。他の人の作品を読む際、**「いいね」**、**「よくないね」**、**「パス」**する権利を有する。読者の主観的判断に影響を与えないため、他の読者による投票結果を見ることはできず、作者のみが見ることができるが、作者自身は投票できない。作者は公開済みのコンテンツを修正する権利を有するため、ユーザーが同じページにアクセスするたびに同じコンテンツが見られる保証はなく、したがって読者はいつでも自身の見解を変更できる。読者はまた、すべてのファイルの履歴バージョンを追跡して閲覧できる。

### **1.2 コンテンツの引用と価値の固定**

* **一意のファイル名：** すべてのコンテンツ断片（物語の章、画像、音声、またはキャラクター、シーン、小道具、種族、魔法、武術、門派、単位、法則、現象、概念、信仰などの定義を含むがこれらに限定されない）は、独立したファイルに保存されなければならず、ファイルは他のファイルにリンクすることができる。
* **明示的引用：** 作者は制作時、**ファイルリンクによる明示的引用**を通じて、自身または他の作者のブランチ内のファイルを参照し、網状構造を形成する。原作者のファイル内に自身のファイルへの参照を追加したい場合は、原作者に申請する必要があり、原作者は同意または拒否できる。mdファイル内での引用形式は `[テキスト](バージョン番号/ユーザー名/作品/パス/ファイル名.拡張子)` であり、バージョン番号は省略可能で、デフォルトは最新バージョンを表示する。
* **価値の定義：** コンテンツの価値は人為的には決定されず、以下の**3つの客観的データ**によってその価値が決定される：**採用パス内の有効文字数、引用回数、純「いいね」数**。

## **2. インセンティブと収益計算**

貢献者が**制作者助成基金**から分配を受ける資格と重みは、**商業化パス内でコンテンツが生み出す客観的データ**に完全に基づいて決定される。

### **2.1 貢献ポイント計算式**

貢献者の総合ポイント（すなわち収益分配の重み）は、以下の**3つの核心データ指標**を加重計算して求める：

貢献ポイント = パス内の全ファイルのポイント合計

単一ファイルポイント = （文字数重み * 基本文字数ポイント + 引用重み * 引用回数） * （当該ファイルの純「いいね」値 / 全ファイルの純「いいね」値）

| 指標 | 価値の代表 | 計算方法 |
| :--- | :--- | :--- |
| **基本文字数ポイント** | **投入された努力（量）** | 貢献者が**最終的に採用された商業化パス内で**提供した有効新規文字数の合計。 |
| **引用回数** | **コンテンツの基礎性（有用性）** | 貢献者のコンテンツが**最終的に採用された商業化パス内で**他の作者に**有効引用された回数の合計**。 |
| **純「いいね」値** | **コミュニティの嗜好性（質）** | 貢献者のコンテンツが**最終的に採用された商業化パス内で**獲得した**純「いいね」数**（「いいね」数 - 「よくないね」数）。 |

### **2.2 動的重みの決定**

**作品理事会は商業化時に協議・投票して重みを決定する。**

1.  **協議：** 今回の収益分配に関与するすべての作者（すなわち、採用パス内にコンテンツがあるすべての作者）はまず協議を行い、各自が提案する重みを提示する。
2.  **投票：** 最終的な重みは、**今回の助成分配に参加するすべての作者**による投票によって決定され、**受益者自治**を実現する。

### **2.3 商業化シナリオと分配**

助成金の分配は、**商業化製品が実際に採用したコンテンツパス**に基づいて行われる：
A, B, C, D の4人の制作者が参加する作品を想定する。

1.  **線形製品（例：映画）：** **最終的に採用された線形の**パス（例：A C D パス）内のすべての作者の貢献ポイントのみを計算する。
2.  **オープン製品（例：オープンワールドゲーム）：** **製品に組み込まれたすべての**貢献者（A, B, C, D）の貢献ポイントを計算する。

## **3. ガバナンスと法的境界**

このモデルは**極小自治**の構造を採用し、作品管理と戦略的意思決定機関として**プラットフォーム理事会**と**作品理事会**を設置する。

### **3.1 プラットフォーム理事会の核心的機能**

1.  **商業化戦略とIP法的リスク**にのみ専念し、白書の策定、DIPCPの開発・アップグレード、および受託者責任を履行する。
2.  CLAの統一管理。
3.  **IP法的リスクの転嫁：** プラットフォーム理事会は、盗用や著作権衝突の認定を**行わない**。このような法的問題が発生した場合、プラットフォーム理事会は**通報情報を公式に記録し関係者に通知する**責任を負うが、法的仲裁や執行には関与せず、**外部の法的手続き**に完全に委ねる。
4.  **コンテンツ規制の最終権限：** DIPCNPO定款第7C条に基づき、プラットフォーム理事会（取締役会）は、75%の絶対多数票を得た場合、「禁止用途」（暴力扇動、搾取的内容など）に該当すると合理的に認定されたコンテンツを削除する最終権限を有し、プラットフォームおよびDIPCNPOの法的存続を保護する。
5.  プロジェクト理事会の協議・投票結果に基づき、**制作者助成基金**から指名された作者に対して助成金を支給する。

### **3.2 作品理事会の核心的機能**

1.  **商業化パス決定：** 作品理事会は、どのように商業化するか、またどのフォークパスを採用して商業化するかを決定する唯一の権限機関である。
2.  **助成指名と投票組織：** 作品理事会は、貢献ポイントとコミュニティ影響力に基づき、高価値な制作者を**制作者助成基金**の助成対象として指名する。理事会メンバーは2つの重みの比率を協議した後、投票を開始する必要があり、有効パス内のすべての作者が投票に参加できる。最終的な助成金受取人および分配額は、プロジェクト範囲内のコミュニティ投票によって決定される。

## **4. 応用例**

### 4.1 基本アルゴリズム
1.  作者Aが発起人となり、**軒轅王朝**についての作品を書いたとする。その中に**張三**と**李四**という二人の人物がいる。すると、これら三つの定義に対してそれぞれ三つのファイルを作成する必要がある。**軒轅王朝**がホームページとなり、**張三**と**李四**のファイルへのリンクを含める必要がある。その後、各章でこの三つの用語が登場するたびに、クリック可能なリンクが自動生成され、1回の引用としてカウントされる。
2.  次に、作者BがAのこの物語を見て面白いと思い、参加を決意する。彼は**王五**というキャラクターを作成し、Aに**王五**のファイルリンクを**軒轅王朝**のファイルに追加するよう依頼する。そして彼は**王五**の伝記を書くが、完成させず、Aもまた後の作品で**王五**というキャラクターを使用する。
3.  次に、作者Cは、結末のない**王五**の物語を見て、彼のために結末を書いてあげる。
4.  一方、作者Dは、Cが書いた結末が良くないと思い、別のバージョンを書く。この結末は予想外に多くの「いいね」を獲得し、映画会社が版権を購入したいと申し出る。
5.  この物語のパスには作者Cが関与していないため、理事会メンバーは**A、B、D**の三人で構成され、**2つの重み**の配分をどのように設定するか協議する。
6.  作品全体で、**軒轅王朝**は100回引用され、文字数は5000語、「いいね」800。**張三**と**李四**はそれぞれ10回引用され、2500語、「いいね」100。**王五**は50回引用され、10万語、「いいね」500。Dが書いた結末は引用されず、2万語だが、「いいね」5000。
7.  すると、Aは当然**引用重み**が高くなることを望み、BとDは**文字数重み**が高くなることを望む。
8.  そこで三人は協議し、決定する：**引用重み** 40%、**文字数重み** 60%。
9.  版権が100万で売れた場合：
- Aのポイントは：
	**軒轅王朝** ((100/160) * 40% + (0.5/13) * 60%) * (800/6500) = 0.0336094674556213
	**張三** ((10/160) * 40% + (0.25/13) * 60%) * (100/6500) = 0.0005621301775147929
	**李四** ((10/160) * 40% + (0.25/13) * 60%) * (100/6500) = 0.0005621301775147929
- Bのポイントは：((50/160) * 40% + (10/13) * 60%) * (500/6500) = 0.0451183431952663
- Dのポイントは：((0/160) * 40% + (2/13) * 60%) * (5000/6500) = 0.0710059171597633
- 合計ポイント：0.1508579881656805
10. 最終的な分配額：
- A = (0.0336094674556213+0.0005621301775147929*2) / 0.1508579881656805 * 1000000 = 230,241.22
- B = 0.0451183431952663 / 0.1508579881656805 * 1000000 = 299,078.25
- C = 0.0710059171597633 / 0.1508579881656805 * 1000000 = 470,680.52
11. その後、このIPがゲーム会社に注目され、すべてのストーリーコンテンツを使用したオープンワールドゲームを開発することになった場合、今回もCは分配に参加し、計算式は上記と同じ。

### 4.2 コンテンツアイランド
1.  作者Bが作者Aの物語を読んだ後、外伝を書いたとする。しかし、Aにリンク申請を提出したが拒否された場合、この記事はコンテンツアイランドに属し、制作プラットフォームの他のユーザーは彼の作品を発見できず、全体のIP価値を計算する際にも含まれない。
2.  しかし、彼は他の経路で記事のリンクを友人と共有できるため、作者CがBの作品を基に別の記事を書き、Bの記事を引用したとする。その後、Cの記事が既にメインネットワークに接続されている作者によって引用されると、Bの記事も間接的にメインネットワークに接続され、それによってIP価値が生じる。

### 4.3 人格分裂
1.  作者Aと作者Bの両方が**張三**という名前のキャラクターを作成し、それぞれのディレクトリに保存したとする。作者CがBの記事しか読んでおらず、後の制作で**張三**を引用する場合、それは当然Bが作成した張三である。
2.  作者DがAとBの両方の記事を読んでいる場合、彼の記事で**張三**に言及する際、人格分裂の状況が発生する。彼は引用するのに適切だと思う人格を選択しなければならない。もちろん、彼は自分で新しい**張三**を作成することもできる。これらの**張三**はすべて、この多元宇宙に同時に存在する。
3.  作者Eが彼ら三人の記事を読み、面白いと思い、三つの**張三**をすべてリンクさせた記事を書き、分岐した世界が再び交差する。

### 4.4 コンテンツ補完
1.  作者Aが記事を書き、その中で通行人（エキストラ）に言及したが、彼のためのキャラクター定義ファイルを作成しなかったとする。作者Bが彼の物語を読み、この通行人が面白いと思い、彼のためにキャラクター定義記事を書く。Aにリンク申請を送り、Aが同意した後、Bの補完コンテンツはネットワーク全体に組み込まれ、以後他の作者が通行人に言及する際、自然とBの記事が引用される。
2.  別のシナリオとして、Aの制作に論理的な欠陥があった。Bがそれを見つけ、その穴を埋める記事を書き、Aに通知する。Aは改訂版でBの内容を追加し、Bの原文にリンクして感謝を表す。将来、文字数、引用回数、純「いいね」数を計算する際、Bの分も含まれる。

### 4.5 盗用判定
記事の内容が盗用に該当するかどうかの判断は非常に難しい。本システムは、これに関する判断と罰則を、コミュニティの市場メカニズムと外部の法的手続きに完全に委ねる。

1.  市場による罰則（コミュニティ自治）：
    証拠と周知： 被害者または読者は、ある記事（またはファイル）の大部分が盗用であることを証明する確実な証拠を提供し、コミュニティに提示する責任を負う。
    罰則メカニズム： コミュニティが盗用コンテンツに対して行う唯一の罰則は市場選択である：大量の「よくないね」を通じて不承認を表明する。これにより、当該ファイルの純「いいね」値が負となり、貢献ポイントがゼロになり、商業化時に採用パスから自動的に除外される。

2.  法的責任と削除：
    法的措置： 著作権、盗用などに関する法的行為は、被害者が外部の執行機関と法的手続きを通じて解決することに完全に依存する。
    理事会の責任： 理事会はコンテンツを強制的に削除する権限はないが、DIPCP上の作品リンクを削除する権利を有する。法的効力のある裁定（裁判所の差し止め命令など）を受領した場合、理事会はすべての作者に通知し、商業化パス決定において、当該ファイルおよびそれが属するパスを永久に除外する。

3.  引用規範：
    複製の例外： 作者Aが**張三**のファイルですでにその容貌、性格などの多くの特徴を詳細に設定しており、作者Bが『張三伝』の中で、筋書きの必要性から、その一部または全部の内容を直接複製したとする。4.4.2のように感謝のリンクを追加する限り、この状況は盗用とはみなされず、ソースコンテンツに対する高価値な引用と見なされ、これにより引用された側（A）に**「引用回数」**のポイント増加がもたらされる。
