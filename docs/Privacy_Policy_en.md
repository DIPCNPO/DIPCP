# Privacy Policy

## DIPCP (Decentralized Intellectual Property Collaboration Platform) Privacy Protection Statement

**Last Updated: December 3, 2025**

---

## Important Notice:

DIPCP is a fully transparent, decentralized IP collaboration platform. This Privacy Policy explains how we handle your information and your privacy rights when using the platform.

## Transparency Principle:

DIPCP adheres to a fully transparent design principle. All data is publicly stored in GitHub repositories, which means your information will be visible to everyone.

---

## 1. Information Handling

### 1.1 We Only Store the CLA (Contributor License Agreement) Signed by Users

DIPCP is a pure client-side application. We only store the CLA (Contributor License Agreement) signed by users and do not collect, store, or process any of your personal information. All data is processed locally on your device. Your GitHub access token will only be uploaded to your personal repository and will never be disclosed to any third party.

### 1.2 Local Data Processing

The application processes the following data only on your device:

- **Local Cache**: Temporary data such as edited content, settings preferences, etc. (local only)
- **GitHub Data**: Public information retrieved via the GitHub API (not stored)
- **User Operations**: Records of actions like editing, saving, etc. (local only)

#### Important Note:

The DIPCP client does **not** send your personal information to any server. All data interacts directly with GitHub services through the GitHub API.

---

## 2. Data Flow

### 2.1 Data Flow Description

The data flow of the DIPCP client application is as follows:

- **User → Local Device**: Data such as edited content and settings preferences are stored on your device.
- **Local Device → GitHub**: Direct interaction with GitHub services via the GitHub API.
- **GitHub → Local Device**: Retrieving public information such as project data, user votes, etc.
- **Us**: We only store the CLA (Contributor License Agreement) signed by users. We do **not** receive, store, or process any user data.

### 2.2 Transparency Guarantee

#### Important Note:

Due to DIPCP's transparent design principle, the following information will be publicly stored:

- Your voting records and created content.

This information will be stored in plain text in GitHub repositories and can be viewed by anyone.

---

## 3. Data Storage

### 3.1 Storage Locations

Data is stored in the following locations:

- **GitHub Repositories**: All project-related data, user information, voting records (managed by GitHub).
- **Your Local Device**: Temporary cache, settings preferences, edit drafts (controlled by you).

### 3.2 Storage Control

- **GitHub Data**: Governed by GitHub's Privacy Policy and Terms of Service.
- **Local Data**: Fully controlled by you and can be deleted at any time.
- **Us**: We only store the CLA (Contributor License Agreement) you signed. We do not store any user data.

---

## 4. Data Sharing

### 4.1 We Do Not Share Your Data

The DIPCP client application does **not** share your data with any third parties, as we do not collect, store, or process any user data.

### 4.2 Data Disclosure Statement

#### Important Reminder:

Due to DIPCP's transparent design, the following information will be publicly stored:

- User works and voting records (stored in GitHub repositories).

This data is managed by GitHub and subject to GitHub's Privacy Policy.

---

## 5. Data Security

### 5.1 Client Security

Security measures of the DIPCP client application:

- Uses HTTPS encryption for communication with the GitHub API.
- Local data is stored on your device, controlled by you.
- Regular updates of application security patches.
- No sensitive information is saved in files.

### 5.2 Data Security Statement

#### Important Reminder:

Due to the transparent design principle, please note the following security limitations:

- Data in GitHub repositories is stored in plain text without encryption protection.
- Anyone can access and view public data.
- Confidentiality of data cannot be guaranteed.
- It is recommended not to upload sensitive or confidential information.

---

## 6. Your Rights

### 6.1 Right to Access

You have the right to:

- View your works and voting records.
- Understand how your data is used and stored.
- Obtain copies of your data and use export functions.

### 6.2 Right to Control

You can:

- Control account information through GitHub settings.
- Delete local cache and temporary data.

### 6.3 Limitations

#### Important Note:

Due to the transparent design principle, the following rights are limited:

- Unable to hide operation history and voting records.
- Unable to retract published articles.
- All changes will be recorded and made public.

---

## 7. Children's Privacy

DIPCP is not specifically designed for children under 13 years of age. If you are a child under 13, please use this platform under the guidance of a parent or guardian.

---

## 8. International Transfers

Your information may be transferred to other countries via GitHub services. We rely on GitHub's data protection measures to safeguard your information.
